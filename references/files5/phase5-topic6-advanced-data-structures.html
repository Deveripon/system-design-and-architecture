Structure 1

Bloom Filter — "আছে কিনা" দ্রুত বলো

Bloom Filter একটা probabilistic data structure যেটা বলতে পারে: "এই element টা definitely নেই" অথবা "এই element টা probably আছে।" কখনো মিথ্যা বলে "নেই" (false negative নেই), কিন্তু মাঝে মাঝে ভুলে বলে "আছে" (false positive হতে পারে)।

🧠 Core Concept
Bloom Filter হলো একটা bit array (শুধু 0 আর 1) এবং কয়েকটা hash function। Element add করতে hash করো, সেই positions গুলো 1 করো। Check করতে আবার hash করো, সব positions 1 হলে "probably exists", কোনোটা 0 হলে "definitely not exists"।
Bloom Filter — Bit Array ও Hash Functions
⚠️ False Positive মানে কী?
Bloom Filter কখনো false negative দেয় না — "নেই" বললে সত্যিই নেই। কিন্তু "আছে" বললে সত্যিই আছে নাও হতে পারে (false positive)। এটা acceptable কারণ false positive rate control করা যায় — bit array বড় করলে কমে।
javascript bloom-filter.js — From Scratch Implementation 
class BloomFilter {
  constructor(size = 1000, hashCount = 3) {
    this.size = size;
    this.hashCount = hashCount;
    this.bitArray = new Uint8Array(size); // সব 0 দিয়ে শুরু
  }

  // Simple hash function — real impl এ MurmurHash3 use করো
  _hash(item, seed) {
    let hash = seed;
    for (let i = 0; i < item.length; i++) {
      hash = (hash * 31 + item.charCodeAt(i)) % this.size;
    }
    return Math.abs(hash);
  }

  // Item add করো — k টা position 1 করো
  add(item) {
    for (let i = 0; i < this.hashCount; i++) {
      const pos = this._hash(item, i * 7919); // different seed per hash
      this.bitArray[pos] = 1;
    }
  }

  // Check করো — সব positions 1? Probably exists. কোনো 0? Definitely not!
  mightContain(item) {
    for (let i = 0; i < this.hashCount; i++) {
      const pos = this._hash(item, i * 7919);
      if (this.bitArray[pos] === 0) return false; // DEFINITELY NOT here
    }
    return true; // PROBABLY here (might be false positive)
  }
}

// Real world use: username check
const takenUsernames = new BloomFilter(100000, 5);
['alice', 'bob', 'charlie', 'david'].forEach(u => takenUsernames.add(u));

// Signup এ check — DB hit এড়ানো যায়
async function checkUsername(username) {
  if (!takenUsernames.mightContain(username)) {
    // DEFINITELY available — DB hit লাগবে না!
    return { available: true, source: 'bloom-filter' };
  }
  // Probably taken — DB confirm করো
  const exists = await db.query('SELECT 1 FROM users WHERE username=?', [username]);
  return { available: !exists, source: 'database' };
}

console.log(takenUsernames.mightContain('alice'));   // true ✓
console.log(takenUsernames.mightContain('eve'));     // false (definitely not)

Structure 2

HyperLogLog — Unique Count Estimate করো

HyperLogLog একটা algorithm যেটা distinct elements এর count (cardinality) estimate করে — মাত্র কয়েক KB memory তে, ৯৯%+ accuracy সহ। "আজকে কতজন unique user এলো?" — HashSet এ billions of user IDs রাখতে হয়, কিন্তু HyperLogLog এ মাত্র ১২ KB।

🧮 Core Idea — Leading Zeros Magic
Random binary string এ leading zeros এর সংখ্যা দিয়ে কতটা strings দেখা হয়েছে estimate করা যায়। যদি তুমি "000001..." দেখেছ (৫টা leading zero), তাহলে সম্ভবত ২^৫ = ৩২টা unique value দেখা হয়েছে। HyperLogLog এই idea কে ৭০+ years এর math দিয়ে refine করে ০.৮১% standard error তে নামিয়ে এনেছে।
HyperLogLog — Leading Zeros থেকে Cardinality Estimate
javascript hyperloglog-demo.js — Redis PFADD / PFCOUNT 
// Production এ Redis এর HyperLogLog use করো
// Redis commands: PFADD (add), PFCOUNT (estimate count)

const redis = require('redis');
const client = redis.createClient();

// Daily unique visitors track করো
async function trackVisitor(userId) {
  const today = new Date().toISOString().split('T')[0]; // "2024-01-15"
  const key = `dau:${today}`;

  // PFADD: userId কে HyperLogLog এ add করো
  // একই userId বারবার add হলেও count একবারই বাড়বে
  await client.pfAdd(key, userId);

  // Key expire করো — ৩০ দিন পর delete হবে
  await client.expire(key, 30 * 24 * 3600);
}

// Daily Active Users count পাও
async function getDailyActiveUsers(date) {
  const key = `dau:${date}`;
  // PFCOUNT: approximate unique count return করে
  const count = await client.pfCount(key);
  return count; // ~0.81% error rate
}

// Weekly Unique Users — multiple HLL merge করা যায়!
async function getWeeklyActiveUsers() {
  const keys = [0,1,2,3,4,5,6].map(d => {
    const date = new Date(Date.now() - d * 86400000);
    return `dau:${date.toISOString().split('T')[0]}`;
  });
  // PFCOUNT multiple keys — automatically merges, deduplicates!
  return client.pfCount(...keys);
}

// Usage simulation
for (let i = 0; i < 1000000; i++) {
  await trackVisitor(`user-${i}`);
}
console.log(await getDailyActiveUsers('2024-01-15')); // ~1000000 (±0.81%)
Structure 3

Count-Min Sketch — Frequency Estimate করো

Count-Min Sketch একটা probabilistic data structure যেটা element এর approximate frequency count রাখে। "এই word টা কতবার দেখা গেছে?" — এই প্রশ্নের উত্তর দেয় massive scale এ, tiny memory তে।

🎯 Core Concept — 2D Counter Array
Count-Min Sketch হলো একটা 2D array of counters (rows = hash functions, columns = counter buckets)। Element add করতে প্রতিটা row এর hash position এ ১ যোগ করো। Count বের করতে সব rows এর minimum নাও — এটাই approximate count।
Count-Min Sketch — 3 Rows × 8 Columns
javascript count-min-sketch.js — Trending Topics 
class CountMinSketch {
  constructor(width = 1000, depth = 5) {
    this.width = width;  // columns
    this.depth = depth;  // rows (= hash functions)
    // 2D array of counters
    this.table = Array.from({ length: depth }, () => new Int32Array(width));
    this.seeds = [7919, 7907, 7901, 7883, 7879]; // prime seeds
  }

  _hash(item, seed) {
    let h = seed;
    for (let c of item) h = (h * 31 + c.charCodeAt(0)) % this.width;
    return Math.abs(h);
  }

  // Event add করো — প্রতিটা row এর hash position increment
  add(item, count = 1) {
    for (let i = 0; i < this.depth; i++) {
      const col = this._hash(item, this.seeds[i]);
      this.table[i][col] += count;
    }
  }

  // Approximate frequency বের করো — minimum নাও
  estimate(item) {
    let min = Infinity;
    for (let i = 0; i < this.depth; i++) {
      const col = this._hash(item, this.seeds[i]);
      min = Math.min(min, this.table[i][col]);
    }
    return min; // approximate count (never underestimates)
  }
}

// Twitter-style trending topics
const trendSketch = new CountMinSketch(10000, 5);

// প্রতিটা tweet process করো
function processTweet(tweet) {
  const words = tweet.toLowerCase().split(/\s+/);
  for (const word of words) {
    trendSketch.add(word);
  }
}

// লক্ষ লক্ষ tweets process করো
tweets.forEach(processTweet);

// Trending check করো
console.log(trendSketch.estimate('cricket'));   // 450000 mentions
console.log(trendSketch.estimate('election'));  // 280000 mentions
console.log(trendSketch.estimate('rain'));       // 15000 mentions
// মাত্র ~50 KB RAM এ সব — exact HashMap এ দরকার হতো GB

Assignment

⏱ সময়: ২-৩ ঘণ্টা 📊 Junior Level

Probabilistic data structures এর concept ভালো করে বোঝার জন্য।

  • Structure Selection: নিচের প্রতিটা problem এর জন্য কোন structure best এবং কেন বলো: (ক) Email spam filter — কোনো email address কি blacklist এ আছে? (খ) YouTube — একটা video কতবার watch করা হয়েছে? (গ) Spotify — একজন user কতটা unique song শুনেছে এই মাসে? (ঘ) E-commerce — product page কি cache এ আছে?
  • False Positive Analysis: Bloom Filter এ bit array size 1MB (8 million bits), hash functions 5টা, elements 1 million। Expected false positive rate বের করার formula হলো (1 - e^(-kn/m))^k যেখানে k=hash count, n=elements, m=bits। Calculate করো। তারপর বলো false positive rate 1% করতে 1M elements এ কত bits লাগবে।
  • Redis HyperLogLog Experiment: Redis install করো (বা Redis Cloud free tier use করো)। Node.js দিয়ে ১০,০০০ random UUID generate করে PFADD করো। PFCOUNT দিয়ে result দেখো। Actual count vs estimated count এর পার্থক্য কত percent?

// ১ মিলিয়ন random strings generate করো
function generateStrings(n) {
  return Array.from({ length: n }, (_, i) => `item-${i}-${Math.random().toString(36)}`);
}

const data = generateStrings(1_000_000);

// Bloom Filter benchmark
const bf = new BloomFilter(10_000_000, 5);
console.time('bloom-add');
data.forEach(s => bf.add(s));
console.timeEnd('bloom-add');

// False positive rate test
let falsePositives = 0;
const testData = generateStrings(10_000); // new strings, not added
testData.forEach(s => { if (bf.mightContain(s)) falsePositives++; });
console.log(`False positive rate: ${(falsePositives / 10_000 * 100).toFixed(2)}%`);

// Count-Min Sketch vs HashMap memory
const words = ['cricket', 'football', 'rain', 'politics'];
const cms = new CountMinSketch(1000, 5);
const exact = new Map();

// 100K random word selections simulate করো
for (let i = 0; i < 100_000; i++) {
  const w = words[Math.floor(Math.random() * words.length)];
  cms.add(w);
  exact.set(w, (exact.get(w) || 0) + 1);
}

words.forEach(w => {
  console.log(`${w}: exact=${exact.get(w)}, approx=${cms.estimate(w)}`);
});
