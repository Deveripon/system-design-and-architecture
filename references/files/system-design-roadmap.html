🔢 Scalability Fundamentals CORE

Vertical vs Horizontal Scaling, এর পার্থক্য, কখন কোনটা ব্যবহার করবে।

কী শিখবে
Vertical scaling (বড় মেশিন) vs Horizontal scaling (বেশি মেশিন)। Load balancer দিয়ে ট্র্যাফিক ভাগ করা। Stateless vs Stateful আর্কিটেকচার।
Use Cases

  • ১০ লক্ষ+ user হলে: horizontal scaling
  • Database bottleneck: vertical first
Practice

  2. Nginx দিয়ে load balance করো
  3. Multiple instances run করো

📦 CAP Theorem & Consistency THEORY

Consistency, Availability, Partition Tolerance — distributed system এর মূল ত্রিভুজ।

CAP Triangle
তিনটার মধ্যে দুটো choose করতে হয়। CA: Traditional SQL। CP: HBase, Zookeeper। AP: Cassandra, DynamoDB। Eventual consistency vs Strong consistency — trade-off বোঝো।
Real Scenarios
  • Bank transfer: Strong consistency (CP)
  • Social media likes: Eventual consistency (AP)
  • DNS: Eventual consistency
Phase 02 Core Components — মূল বিল্ডিং ব্লক ৬-৮ সপ্তাহ ▼
⚡ Caching Strategy HIGH IMPACT

Cache এর সঠিক ব্যবহারে system ১০-১০০x ফাস্ট হয়। এটা না জানলে চলবে না।

Cache Patterns
  • Cache-Aside (Lazy loading): সবচেয়ে common
  • Write-Through: data loss কম, latency বেশি
  • Write-Back: fast write, data loss risk
  • Read-Through: cache layer handles DB
Eviction Policies
LRU (Least Recently Used) — সবচেয়ে বেশি used। LFU (Least Frequently Used)। TTL (Time to Live) — expiry set করো।
Tools
Redis Memcached CDN (CloudFront) Varnish
Practice
  1. Redis install করো, simple key-value করো
  2. User profile cache করো TTL=1hr
  3. Cache hit rate measure করো
  4. Cache stampede কী সেটা handle করো
⚖️ Load Balancing HIGH IMPACT

Traffic কীভাবে distribute করবে — algorithms, health checks, sticky sessions।

Algorithms
  • Round Robin: সমান ট্র্যাফিক, simple
  • Least Connections: busy server কম load
  • IP Hash: same user → same server
  • Weighted: powerful server বেশি load
L4 vs L7
L4 (Transport): TCP/UDP level। L7 (Application): HTTP header/URL দেখে route করে। L7 বেশি intelligent কিন্তু overhead বেশি।
Tools
Nginx HAProxy AWS ALB/NLB Cloudflare
📨 Message Queues & Event Streaming MUST KNOW

Async communication, decoupling services — modern distributed system এর মেরুদণ্ড।

Queue vs Stream
Queue (RabbitMQ): একবার consume হলে delete। Stream (Kafka): data retain করে, replay possible। Queue = task processing। Stream = event sourcing, analytics।
Use Cases

  • Real-time analytics: Kafka
  • Order processing: SQS/RabbitMQ
  • Log aggregation: Kafka
Tools
Apache Kafka RabbitMQ AWS SQS/SNS Redis Streams
Practice

  2. Producer → Consumer বানাও

  4. Dead letter queue implement করো

💾 Database Sharding & Replication ADVANCED

Large-scale database management — horizontal partitioning এবং data redundancy।

Sharding Strategies
  • Range-based: user_id 1-1M → shard 1
  • Hash-based: consistent hashing
  • Directory-based: lookup table
  • Geo-based: region অনুযায়ী
Replication
Master-Slave: write → master, read → slave। Master-Master: both write। Multi-region replication: latency কমাতে। Read replica দিয়ে read load handle করো।
Hotspot Problem
Consistent hashing দিয়ে hotspot avoid করো। Celebrity problem: viral user এর data একই shard এ → Virtual nodes ব্যবহার করো।
🗂️ CDN & Object Storage PRACTICAL

CDN কীভাবে কাজ করে
Edge servers globally ছড়িয়ে থাকে। User কাছের edge থেকে content পায়। Origin server load কমে। Cache-Control headers দিয়ে TTL set করো।
Tools
AWS S3 CloudFront Cloudflare MinIO (self-hosted)

🧩 Microservices Architecture ARCHITECTURE

Monolith থেকে Microservices এ যাওয়ার পথ, trade-offs, service mesh।

Monolith vs Microservices
  • Startup: Monolith দিয়ে শুরু করো
  • ১০০+ developers হলে: Microservices
  • Independent deploy দরকার: Microservices
  • Simple app: Monolith ভালো
Service Communication
Synchronous: REST/gRPC — simple কিন্তু tight coupling। Asynchronous: Kafka/RabbitMQ — loose coupling, complex। Service mesh (Istio/Linkerd): service discovery, load balancing, mTLS।
Tools
Docker Kubernetes Istio Consul

🔄 Circuit Breaker & Resilience RELIABILITY

Cascading failures থামানো — Circuit Breaker, Retry, Timeout, Bulkhead patterns।

Resilience Patterns
  • Circuit Breaker: failed service isolate করো
  • Retry with Exponential Backoff
  • Timeout: hanging request kill করো
  • Bulkhead: resource pool isolate করো
  • Fallback: default response দাও
Tools
Resilience4j Hystrix (legacy) Polly (.NET)
Netflix এর উদাহরণ
Recommendation service down হলেও homepage চলে। Circuit open হলে cached/default content দেখায়। Graceful degradation = user experience বাঁচানো।
🌊 Rate Limiting & Throttling SECURITY

DDoS protection, fair usage — Token Bucket, Leaky Bucket, Sliding Window algorithms।

Algorithms
  • Token Bucket: burst traffic allow করে
  • Leaky Bucket: steady rate enforce করে
  • Fixed Window Counter: simple কিন্তু edge case আছে
  • Sliding Window Log: সবচেয়ে accurate
Implementation
Redis দিয়ে distributed rate limiting করো। API Gateway তে global limit। Per-user, per-IP, per-endpoint — granularity choose করো।
Tools
Redis + Lua Kong Rate Limiting Cloudflare WAF

🔁 Event-Driven & CQRS/Event Sourcing ADVANCED

Command Query Responsibility Segregation, Event Sourcing — enterprise pattern।

CQRS
Read এবং Write আলাদা model রাখো। Write: normalized DB (consistency)। Read: denormalized/cached (performance)। Complex domain এ খুব কার্যকর।
Event Sourcing
State save করো না, events save করো। State = events replay করার result। Audit trail free পাও। Time travel debugging possible।
Use Cases
  • Banking: transaction history
  • E-commerce: order state machine
  • Audit-heavy systems

🎯 Interview এ System Design করার Framework

1. Requirements (5 min)

Functional requirements কী? Non-functional: scale, latency, availability। Clarify করো, assume করো না।

2. Estimation (3 min)

DAU, QPS, storage estimate করো। Back-of-envelope calculation। Read/Write ratio বলো।

3. High-Level Design (10 min)

বড় components আঁকো। Client → LB → Services → DB। Data flow explain করো।

4. Deep Dive (15 min)

Interviewer যেটায় focus করতে বলে সেটা detail এ বলো। Trade-offs discuss করো।

5. Scale করো

Bottlenecks identify করো। Cache কোথায়? Sharding কোথায়? Async কোথায়?

6. Wrap Up

Monitoring কীভাবে করবে? Failure scenarios কী? Further improvements কী কী?
