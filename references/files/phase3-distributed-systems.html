Topic 01

🧩 Microservices Architecture

Microservices মানে একটা বড় app কে ছোট ছোট independent service এ ভাগ করা যেগুলো নিজেদের মধ্যে communicate করে। প্রতিটা service একটাই কাজ ভালো করে।

🎯 Industry Advice
Martin Fowler বলেন: "Don't start with microservices." Monolith দিয়ে শুরু করো, boundaries বোঝো, তারপর microservices এ extract করো। Amazon, Netflix সবাই monolith দিয়ে শুরু করেছিলো।

Service Communication Patterns

🔗 Synchronous (REST/gRPC)

Service A → Service B কে directly call করে, response আসা পর্যন্ত wait করে। Simple কিন্তু tight coupling। B slow হলে A ও slow।

📨 Asynchronous (Queue/Event)

Service A message পাঠায়, wait করে না। B নিজের pace এ process করে। Loose coupling, resilient কিন্তু complex debugging।

Service Mesh — Istio/Linkerd

Microservices এ প্রতিটা service-to-service call এ auth, retry, timeout, circuit breaking লাগে। এটা প্রতিটা service এ লেখা DRY violation। Service Mesh এটা infrastructure level এ handle করে।

📌 Service Mesh Features
mTLS: Service-to-service encrypted এবং authenticated communication
Load Balancing: Smart L7 load balancing
Circuit Breaking: Automatic failure detection
Observability: Automatic tracing, metrics
Traffic Management: Canary deployment, A/B testing

Topic 02

🔒 Distributed Consensus & Coordination

Distributed system এ সবচেয়ে কঠিন সমস্যা — কীভাবে নিশ্চিত করবে সব nodes একমত? Network partition হলে কী করবে? Leader down হলে কে নতুন leader হবে?

Raft Algorithm — Leader Election

📌 Raft কীভাবে কাজ করে
States: প্রতিটা node হয় Follower, Candidate, বা Leader।
Election: Leader heartbeat না পেলে Follower → Candidate। Vote request পাঠায়। Majority vote পেলে Leader।
Log Replication: Leader সব writes নেয়, Followers এ replicate করে। Majority acknowledge করলে committed।
Split Brain Prevention: Majority quorum দরকার — odd number nodes রাখো (3, 5, 7)।

Distributed Lock — Concurrent access prevent

Pythondistributed_lock.py
import redis, uuid, time

class DistributedLock:
    def __init__(self, redis_client, key, ttl=30):
        self.r = redis_client
        self.key = f"lock:{key}"
        self.ttl = ttl
        self.token = str(uuid.uuid4())
    
    def acquire(self, timeout=10):
        """Lock acquire করার চেষ্টা করো"""
        deadline = time.time() + timeout
        while time.time() < deadline:
            # SET NX = only if not exists (atomic)
            acquired = self.r.set(
                self.key, self.token,
                nx=True, ex=self.ttl
            )
            if acquired:
                return True
            time.sleep(0.1)
        return False
    
    def release(self):
        """Lua script — atomic check and delete"""
        script = """
        if redis.call('get', KEYS[1]) == ARGV[1] then
            return redis.call('del', KEYS[1])
        else return 0 end
        """
        self.r.eval(script, 1, self.key, self.token)

# Usage
lock = DistributedLock(r, "payment:user:123")
if lock.acquire():
    try:
        process_payment()
    finally:
        lock.release()

Topic 03

🔄 Circuit Breaker & Resilience Patterns

একটা service fail হলে সেটা যেন পুরো system না ভাঙে — এটাই resilience। Cascading failure মানে একটা service down হয়ে chain reaction এ সব service down।

Circuit Breaker — Three States

Pythoncircuit_breaker.py
import time
from enum import Enum

class State(Enum):
    CLOSED = "closed"
    OPEN = "open"
    HALF_OPEN = "half_open"

class CircuitBreaker:
    def __init__(self, failure_threshold=5, recovery_timeout=60):
        self.state = State.CLOSED
        self.failures = 0
        self.threshold = failure_threshold
        self.recovery_timeout = recovery_timeout
        self.last_failure = None
    
    def call(self, func, *args):
        if self.state == State.OPEN:
            # Check করো timeout হয়েছে কিনা
            if time.time() - self.last_failure > self.recovery_timeout:
                self.state = State.HALF_OPEN
            else:
                raise Exception("Circuit OPEN — service unavailable")
        
        try:
            result = func(*args)
            self.on_success()
            return result
        except Exception as e:
            self.on_failure()
            raise e
    
    def on_success(self):
        self.failures = 0
        self.state = State.CLOSED
    
    def on_failure(self):
        self.failures += 1
        self.last_failure = time.time()
        if self.failures >= self.threshold:
            self.state = State.OPEN

Resilience Patterns — সব একসাথে

⏰ Timeout

কোনো request কত সময় wait করবে সেটা define করো। Service hang করলে resource free হয়। Default timeout সবসময় set করো।

🔁 Retry with Backoff

Fail হলে retry করো। কিন্তু immediately না — exponential backoff: 1s, 2s, 4s, 8s। Jitter যোগ করো।

🧱 Bulkhead

Resource pool isolate করো। Payment service এর জন্য আলাদা thread pool — এটা fail হলে অন্য services affected হয় না।

🔙 Fallback

Service fail হলে default/cached response দাও। Netflix: recommendation fail → popular movies দেখাও।

Pythonretry_backoff.py
import time, random

def retry_with_backoff(func, max_retries=3, base_delay=1):
    for attempt in range(max_retries):
        try:
            return func()
        except Exception as e:
            if attempt == max_retries - 1:
                raise e
            # Exponential backoff + jitter
            delay = base_delay * (2 ** attempt)
            jitter = random.uniform(0, delay * 0.1)
            time.sleep(delay + jitter)
            print(f"Retry {attempt+1} after {delay:.1f}s")

Python + Redistoken_bucket.py
import time

class TokenBucket:
    def __init__(self, capacity=100, refill_rate=10):
        # capacity: max burst, refill_rate: tokens/second
        self.capacity = capacity
        self.refill_rate = refill_rate
    
    def allow_request(self, user_id):
        key = f"bucket:{user_id}"
        now = time.time()
        
        # Lua script — atomic operation
        lua_script = """
        local key = KEYS[1]
        local capacity = tonumber(ARGV[1])
        local refill_rate = tonumber(ARGV[2])
        local now = tonumber(ARGV[3])
        
        local data = redis.call('HMGET', key, 'tokens', 'last_refill')
        local tokens = tonumber(data[1]) or capacity
        local last_refill = tonumber(data[2]) or now
        
        -- Refill tokens
        local elapsed = now - last_refill
        tokens = math.min(capacity, tokens + elapsed * refill_rate)
        
        if tokens >= 1 then
            tokens = tokens - 1
            redis.call('HMSET', key, 'tokens', tokens, 'last_refill', now)
            redis.call('EXPIRE', key, 3600)
            return 1  -- allowed
        else
            return 0  -- blocked
        end
        """
        result = r.eval(lua_script, 1, key, self.capacity, self.refill_rate, now)
        return result == 1

Topic 05

🔑 Service Discovery

Microservices এ প্রতিটা service dynamically scale করে। IP address বদলে যায়। Service A কীভাবে জানবে Service B কোথায় আছে?

Client-Side vs Server-Side Discovery

👤 Client-Side Discovery

Client নিজেই Service Registry থেকে available instances জানে এবং load balance করে। Example: Netflix Eureka + Ribbon।

🖥️ Server-Side Discovery

Client Load Balancer কে call করে। LB Registry চেক করে route করে। Client কিছু জানে না। Example: AWS ALB + ECS।

Pythonservice_registry.py
import consul

c = consul.Consul()

# Service register করো (startup এ)
c.agent.service.register(
    name='user-service',
    service_id='user-service-1',
    address='10.0.0.5',

    check=consul.Check.http('http://10.0.0.5:8080/health', interval='10s')

# Service discover করো (request করার সময়)
_, services = c.health.service('user-service', passing=True)
instances = [(s['Service']['Address'], s['Service']['Port']) for s in services]
# Round robin বা random select করো

Topic 06

📋 Saga Pattern — Distributed Transactions

Microservices এ একটা business transaction multiple services span করে। Payment, Inventory, Shipping — তিনটাই succeed করতে হবে। একটা fail হলে rollback করতে হবে। Traditional 2PC কাজ করে না। এই সমস্যার সমাধান Saga Pattern।

Choreography vs Orchestration

💃 Choreography

কোনো central controller নেই। প্রতিটা service event publish করে, অন্যরা subscribe করে react করে। Loose coupling কিন্তু debug করা কঠিন।

🎼 Orchestration

Central Saga Orchestrator সব steps পরিচালনা করে। কোন service কখন কী করবে orchestrator জানে। Debug সহজ কিন্তু central component।

Pythonsaga_orchestrator.py
class OrderSagaOrchestrator:
    async def execute(self, order):
        saga_log = []
        
        try:
            # Step 1: Payment
            await payment_service.charge(order.user_id, order.amount)
            saga_log.append('payment')
            
            # Step 2: Inventory reserve
            await inventory_service.reserve(order.items)
            saga_log.append('inventory')
            
            # Step 3: Shipping schedule
            await shipping_service.schedule(order)
            saga_log.append('shipping')
            
            return "SUCCESS"
            
        except Exception as e:
            # Compensating transactions — reverse order
            if 'inventory' in saga_log:
                await inventory_service.release(order.items)
            if 'payment' in saga_log:
                await payment_service.refund(order.user_id, order.amount)
            raise e

প্রশ্ন 02

A সব nodes এ same data রাখো
B Odd number of nodes রাখো (3, 5, 7) যাতে majority quorum সম্ভব হয়

সঠিক: B — Split brain: network partition এ দুটো group নিজেদের leader মনে করে। Majority quorum (N/2 + 1): 3 nodes → 2 majority। Partition হলে smaller group commit করতে পারবে না। Even number = tie possible।

Phase 3 — Project: Resilient Microservices

Circuit Breaker এবং Rate Limiting সহ ২টা microservices বানাও।

1

দুটো Service

User Service (port 8001) এবং Product Service (port 8002)। FastAPI দিয়ে।

2

Circuit Breaker যোগ করো

User Service থেকে Product Service call করো। Product Service artificially slow করো (time.sleep(5))। Circuit Breaker দিয়ে ৩ failures এর পর open করো।

3

Rate Limiting

Redis দিয়ে per-user rate limiting। Free: 10 req/min, Paid: 100 req/min।

4

Test করো

Product Service kill করো। দেখো Circuit Breaker open হচ্ছে কিনা। Fallback response আসছে কিনা।

5

Logs দেখো

Circuit state transitions log করো। Rate limit hits log করো। Pattern বোঝো।
