01 — Core Concept

CDN ও Object Storage কী?

ধরো তুমি ঢাকায় বসে Netflix এ একটা movie দেখছো। সেই movie র file কি USA এর server থেকে আসছে? না! Netflix CDN এর মাধ্যমে সেই file টা তোমার কাছাকাছি একটা server এ (হয়তো Singapore বা Mumbai তে) cache করে রাখে। তুমি সেখান থেকেই পাচ্ছো — ১০x faster।

💡 দুটো Key Concept
CDN (Content Delivery Network): বিশ্বের বিভিন্ন স্থানে PoP (Point of Presence) servers রেখে static content (images, videos, CSS, JS) কাছে থেকে serve করে। Latency কমায়।

Object Storage (S3-like): Files, images, videos কে key-value store এ রাখা। Unlimited storage, highly durable (99.999999999% — 11 nines!), cheap। Database এর মতো query করা যায় না কিন্তু files store এর জন্য perfect।
📊 CDN ছাড়া vs CDN সহ
ঢাকা → USA server: ~200-300ms latency। ঢাকা → Singapore CDN PoP: ~30-50ms latency। ৫-১০x faster! Netflix, YouTube, Facebook সবাই CDN use করে। CDN ছাড়া global scale impossible।
02 — CDN How it Works

CDN কীভাবে কাজ করে?

CDN এর মূল concept হলো Edge Caching। User এর request প্রথমে nearest CDN edge server এ যায়। Cache hit হলে সেখান থেকেই response। Miss হলে origin server থেকে content fetch করে cache করে রাখে।

CDN Cache Control

কতক্ষণ CDN এ cache থাকবে সেটা Cache-Control header দিয়ে control করা যায়।

HTTP Headerscache-control.txt

# Private (user-specific) — CDN cache করবে না

# Browser cache করতে পারে, CDN পারবে না

03 — Object Storage (S3)

AWS S3 — Object Storage

S3 (Simple Storage Service) হলো AWS এর object storage। File upload করো, একটা URL পাও — ব্যস। Database এর মতো complex — না, শুধু key-value: key = file path, value = file content।

🗂️ S3 Concepts
Bucket: Files এর container (folder এর মতো)। Globally unique name। Object: একটা file + metadata। Key: Object এর path/name। Storage Class: Standard (frequent access), Glacier (archival, cheap), Intelligent-Tiering (auto)।
✅ S3 ব্যবহার করো

User uploaded files (images, videos, documents), Static website hosting, Backup and archive, Application logs, ML training data।

❌ S3 ব্যবহার করো না

Frequently updated structured data (database use করো), Real-time queries needed, Strong consistency needed (S3 eventually consistent ছিল, now strong), File system operations (rename, move costly)।

S3 Storage Classes

ClassAccessCostUse Case
S3 StandardInstantHighestFrequently accessed — profile photos, app assets
S3-IAInstant30% cheaperInfrequent access — backups, older files
S3 GlacierMinutes/hoursVery cheapLong-term archive — compliance, audit logs
Glacier Deep12 hoursCheapest7-10 year retention, rarely accessed

💰 Cost Saving Tip
S3 Lifecycle Policy দিয়ে auto-transition করো: প্রথম ৩০ দিন Standard, ৩০-৯০ দিন S3-IA, ৯০+ দিন Glacier। এতে automatically cost কমে। Instagram এই approach এ petabytes of photos manage করে।
04 — CloudFront

AWS CloudFront — CDN

CloudFront হলো AWS এর CDN। S3 এর সাথে perfectly integrate হয়। S3 bucket কে origin set করো, CloudFront globally distribute করবে।

✅ CloudFront Key Features
Origin Shield: Origin এর সামনে extra caching layer — origin এ আরো কম requests। Lambda@Edge: CDN edge এ code run করো — geolocation, A/B testing, auth। Signed URLs: Private content এ time-limited access। WAF Integration: DDoS এবং bot protection।

🔑 Presigned URL — কেন Important?
User file upload করলে API server এর bandwidth waste হবে। Presigned URL দিলে user সরাসরি S3 তে upload করে। API server শুধু temporary signed URL generate করে — নিজে file handle করে না। Security maintained, bandwidth saved।
06 — Code Examples

হাতে-কলমে Code

Python — S3 Upload & Presigned URL

Pythons3_operations.py
import boto3
from botocore.exceptions import NoCredentialsError

s3 = boto3.client(
    's3',
    aws_access_key_id='YOUR_KEY',
    aws_secret_access_key='YOUR_SECRET',
    region_name='ap-southeast-1'  # Singapore
)

# ── File Upload ────────────────────────────────
def upload_file(local_path, bucket, s3_key):
    try:
        s3.upload_file(
            local_path, bucket, s3_key,
            ExtraArgs={
                'ContentType': 'image/webp',
                'CacheControl': 'public, max-age=31536000',
                'ACL': 'public-read'
            }
        )
        print(ff"✅ Uploaded to s3://{bucket}/{s3_key}")
        return ff"https://cdn.myapp.com/{s3_key}"
    except NoCredentialsError:
        raise Exception("AWS credentials not found")

# ── Presigned URL — Direct Browser Upload ──────
def generate_presigned_url(bucket, s3_key, expiry=3600):
    url = s3.generate_presigned_url(
        ClientMethod='put_object',
        Params={
            'Bucket': bucket,
            'Key': s3_key,
            'ContentType': 'image/jpeg'
        },
        ExpiresIn=expiry  # 1 hour
    )
    print(f"⏰ Presigned URL expires in {expiry}s")
    return url

# ── Lifecycle Policy — Auto archive ────────────
s3.put_bucket_lifecycle_configuration(
    Bucket='my-app-uploads',
    LifecycleConfiguration={
        'Rules': [{
            'ID': 'auto-archive',
            'Status': 'Enabled',
            'Transitions': [
                {'Days': 30, 'StorageClass': 'STANDARD_IA'},
                {'Days': 90, 'StorageClass': 'GLACIER'},
            ]
        }]
    }
)

Node.js — Express + S3 Upload API

Node.jsupload_api.js
const { S3Client, PutObjectCommand } = require('@aws-sdk/client-s3');
const { getSignedUrl } = require('@aws-sdk/s3-request-presigner');
const { v4: uuidv4 } = require('uuid');

const s3 = new S3Client({ region: 'ap-southeast-1' });

// Generate presigned URL for direct browser upload
app.post('/api/upload-url', async (req, res) => {
  const { fileType, fileName } = req.body;

  // Unique key prevent overwrite
  const key = `uploads/${uuidv4()}-${fileName}`;

  const command = new PutObjectCommand({
    Bucket: 'my-app-bucket',
    Key: key,
    ContentType: fileType,
  });

  // URL expires in 15 minutes
  const uploadUrl = await getSignedUrl(s3, command, { expiresIn: 900 });
  const publicUrl = `https://cdn.myapp.com/${key}`;

  res.json({ uploadUrl, publicUrl, key });
  // Frontend: PUT uploadUrl সরাসরি, তারপর publicUrl save করো
});
07 — Interview Prep

Common Interview Questions

❓ Q1: CDN ব্যবহার না করলে কী হবে?
উত্তর: Global users কে origin server থেকে serve করতে হবে। USA → Bangladesh: ~250ms latency। CDN সহ: Singapore PoP → ~35ms। ৭x slow। Large files (videos) এ এটা catastrophic। Origin server এ extra load।
❓ Q2: CDN Cache Invalidation কীভাবে করবে?
উত্তর: ৩টা approach: (১) Versioned filenames — logo.v2.png (সবচেয়ে reliable)। (২) CloudFront Invalidation API — নির্দিষ্ট paths invalidate করো (কিছুটা delay)। (৩) Short TTL — frequent updates এর files এ কম TTL দাও।
❓ Q3: S3 vs Database — কখন কোনটা?
উত্তর: S3 — Binary files (images, video, PDF), Large objects, No complex queries, High durability cheap। Database — Structured data, Complex queries (JOIN, filter), Transactions needed, Frequently updated। Instagram: Photos → S3, User data/likes/comments → PostgreSQL।
❓ Q4: Presigned URL কেন ব্যবহার করবে?
উত্তর: File upload এ API server কে bypass করা। Client সরাসরি S3 এ upload করে → API server এর bandwidth ও CPU save। Time-limited URL (15 mins expire) → secure। Server scaling করতে হয় না শুধু uploads এর জন্য।

Question 02 / 10
S3 Object Storage কোন ধরনের data store এর জন্য সবচেয়ে উপযুক্ত?
A User profile data (name, email, age)
B Real-time transaction data
C Images, videos, documents, backups

সঠিক উত্তর: C — S3 binary files এর জন্য perfect: unlimited storage, highly durable, cheap। Complex queries করা যায় না, কিন্তু files store এবং serve এর জন্য ideal।
Question 03 / 10
CDN Cache-Control: max-age=31536000 মানে কী?
A ১ বছর (31,536,000 seconds) CDN এ cache থাকবে
B ৩১ দিন cache থাকবে
C ৩১ মিলিয়ন requests serve করবে
D ৩১ MB পর্যন্ত file cache হবে
সঠিক উত্তর: A — max-age value seconds এ। 31,536,000 = 365 days = 1 year। Static assets (CSS, JS, images) এ এটা common — কিন্তু তখন file এর নামে hash দিতে হয় (app.abc123.js) যাতে update করলে নতুন URL হয়।
Question 04 / 10
Presigned URL কেন ব্যবহার করা হয়?
A Files encrypt করতে
B URL shorten করতে
C CDN bypass করতে
D Client সরাসরি S3 এ upload করতে পারে — API server bandwidth waste হয় না
সঠিক উत্তর: D — Presigned URL একটা temporary signed URL যা client কে সরাসরি S3 এ upload করতে দেয়। API server টা শুধু URL generate করে — file দেখে না। API server এর bandwidth ও scaling সমস্যা কমে।
Question 05 / 10
S3 Glacier কোন use case এর জন্য?

C Real-time video streaming
D Session storage
সঠিক উত্তর: B — S3 Glacier archival storage। Retrieval time minutes to hours। কিন্তু Standard এর তুলনায় ৮০%+ সস্তা। Compliance logs, old backups, audit records এর জন্য perfect।
Question 06 / 10

সঠিক উত্তর: C — Cache invalidation approach: CloudFront Invalidation API দিয়ে specific paths purge করো। অথবা file এ version/hash add করো (logo.v2.png) — নতুন URL মানে CDN এ নতুন entry।
Question 07 / 10
